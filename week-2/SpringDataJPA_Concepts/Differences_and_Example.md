# Difference between JPA, Hibernate and Spring Data JPA

1. **JPA (Java Persistence API)**: 
   - JPA is a specification or a set of rules (interfaces) in Java for object-relational mapping (ORM). 
   - It defines how Java objects map to relational database tables, but it does not provide any implementation itself. It is just an API.

2. **Hibernate**:
   - Hibernate is a popular ORM framework that provides the actual implementation of the JPA specification.
   - While JPA defines the "what" (the interface), Hibernate provides the "how" (the implementation). You can use Hibernate without JPA, but using Hibernate as a JPA provider is the industry standard.

3. **Spring Data JPA**:
   - Spring Data JPA is a part of the larger Spring Data family that adds an extra layer of abstraction on top of a JPA provider (like Hibernate).
   - It significantly reduces boilerplate code required to implement data access layers (DAOs/Repositories) by automatically generating repository implementations based on interfaces (e.g., `JpaRepository`).

# Spring Data JPA - Quick Example

Here is a quick example showing how Spring Data JPA reduces boilerplate code:

### 1. The Entity
```java
import javax.persistence.Entity;
import javax.persistence.Id;

@Entity
public class Employee {
    @Id
    private Long id;
    private String name;
    private String department;
    
    // Getters and Setters
}
```

### 2. The Repository Interface (Spring Data JPA)
By simply extending `JpaRepository`, Spring Data JPA automatically provides implementations for common operations like `save()`, `findAll()`, `findById()`, and `deleteById()`.
```java
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface EmployeeRepository extends JpaRepository<Employee, Long> {
    // Custom query method generated by Spring Data JPA just by method signature!
    List<Employee> findByDepartment(String department);
}
```

### 3. Usage in a Service
```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
public class EmployeeService {

    @Autowired
    private EmployeeRepository employeeRepository;

    public void saveEmployee(Employee emp) {
        employeeRepository.save(emp);
    }

    public List<Employee> getITEmployees() {
        return employeeRepository.findByDepartment("IT");
    }
}
```
